# Troubleshooting EPERM Errors on Windows

## Проблема
Next.js може видавати помилки `EPERM: operation not permitted` на Windows при роботі з файлами в папці `.next`.

## Рішення

### 1. Швидке рішення
```bash
npm run clean
npm run dev
```

### 2. Додати папку `.next` в винятки антивірусу

#### Windows Defender:
1. Відкрийте "Вірус та загрози" в налаштуваннях Windows
2. Натисніть "Керування налаштуваннями" під "Захист від вірусів та загроз"
3. Прокрутіть до "Винятки" і натисніть "Додати або видалити винятки"
4. Натисніть "Додати виняток" → "Папка"
5. Виберіть папку `.next` у вашому проєкті

#### Для інших антивірусів:
Додайте папку `.next` в винятки сканування в налаштуваннях вашого антивірусу.

### 3. Запуск з правами адміністратора
Якщо проблема продовжується, спробуйте запустити термінал/IDE з правами адміністратора.

### 4. Перевірка заблокованих файлів
Використовуйте утиліту [Process Explorer](https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer) для перевірки, який процес блокує файли.

### 5. Альтернативний спосіб запуску
```bash
# Використовуйте турбо режим (якщо доступно)
npm run dev -- --turbo
```

## Причини
- Windows Defender або інший антивірус сканує файли під час їх створення
- Індексер Windows блокує файли
- Недостатньо прав доступу до папки
- Файли відкриті в іншій програмі

## Профілактика
1. Додайте `.next` в винятки антивірусу
2. Використовуйте `npm run clean` перед запуском, якщо є проблеми
3. Переконайтеся, що всі екземпляри dev сервера зупинені перед запуском нового
